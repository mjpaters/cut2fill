# Dev Journal — 2026-04-08

## 10:00 — HERE Routing API integrated into logistics calculator

Replaced haversine + ROAD_FACTOR estimate with live HERE Routing API v8 calls in the delivery estimator panel.

### What changed
- `app.js`: Added `HERE_API_KEY`, `routeCache`, `currentRouteData` module-level vars
- `fetchHereRoute()`: async function — calls HERE Routing API v8 with truck params (42.5t, 430cm height, 19m length), caches result by lat/lng pair
- `decodeFlexPolyline()`: HERE flexible polyline decoder (ported from prototype)
- `showRoutePolyline()`: draws the actual road route on the map (replaces straight dashed line)
- `calcLogistics()`: now accepts optional `durationSec` — uses real HERE travel time when available, falls back to AVG_SPEED_KMH estimate
- `updateCalcResults()`: uses `currentRouteData` when available, else haversine × ROAD_FACTOR fallback. Footer note shows "· estimated" when falling back.
- `showFacilityPanel()`: resets `currentRouteData`, renders panel with estimate immediately, then async-fetches real route and updates distance badge, polyline, and calc results
- `buildLogisticsPanel()`: distance badge now has `id="routeDistanceBadge"` so it can be updated post-fetch

### Why
The NHVR overlay used in the prototype was for B-doubles and road trains (permit vehicles). The actual Cut2Fill use case is body trucks, truck & dogs, and semi tippers — standard class vehicles. HERE's truck routing handles their real constraints (bridge heights, weight limits, built-up area curfews) natively via the API. The haversine × 1.35 estimate was often significantly wrong for routes with detours, rivers, or motorway constraints.

### Truck params used
Single set of params for all three vehicle types (42.5t / 430cm / 19m). Routes don't meaningfully differ between classes in SEQ — the restrictions that matter for B-doubles (permit networks) don't apply here. One API call per unique source-destination pair, cached.

### Fallback behaviour
If HERE API call fails (network, quota, etc.), panel silently falls back to haversine estimate with "· estimated" label. No broken UI.
