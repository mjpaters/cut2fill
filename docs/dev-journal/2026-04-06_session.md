## 16:30 — Heavy vehicle route planning research
- Deep research into QLD/Australian heavy vehicle routing ecosystem for potential Cut2Fill integration
- Key findings: NHVR has a developer portal (api-portal.nhvr.gov.au) with REST APIs including spatial data and a Network Find endpoint returning GeoPackage files per vehicle class
- QLD open data: heavy vehicle routes + restrictions datasets available as SHP/TAB/FGDB/KMZ/GPKG — BUT gazetted routes transitioned to NHVR National Network Map as of April 2024, QLD data now static
- NHVR Spatial Open Data hub (spatialopendata-nhvr.hub.arcgis.com) offers GeoJSON, CSV, KML downloads plus WMS/WFS/GeoServices APIs
- Bridge data: TMR bridges dataset exists (SHP/FGDB) but attribute detail unclear; BCC publishes low-clearance bridges (<4.6m) as open data
- HERE Routing API v8 is the strongest integration candidate: truck routing with height/weight/width/length/axle weight params, Australia coverage confirmed (incl. weight-per-axle data), Leaflet plugin exists (leaflet-routing-machine-here_v8), free tier = 5,000 truck routing transactions/month then $2.50/1000
- Google Routes API does NOT support truck routing — no heavy vehicle parameters at all
- TomTom has truck routing for 22 countries incl. Australia but only major/interconnecting roads
- OpenRouteService is free/open-source with HGV profile (height/weight/axle/hazmat) but relies on OSM data quality
- Gap analysis: NHVR tool is regulatory/compliance only, not turn-by-turn navigation. Fragmented across 6 states, 2 territories, hundreds of councils. No platform combines routing + live closures + material sourcing
- HVAMS (Heavy Vehicle Access Management System) is the future national system — generates tailored access maps per vehicle in seconds, but no public API yet
- QLDTraffic GeoJSON API provides live road closures/hazards/flooding as open data — could overlay on Cut2Fill map
- Decision: HERE API + NHVR open spatial data + QLDTraffic live feed = viable stack for intelligent truck routing in Cut2Fill
